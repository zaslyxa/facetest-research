# План реализации Facetest

## Цель

Сделать небольшую веб-программу для исследования узнавания лиц:

- участник открывает ссылку;
- заполняет анкету: имя, возраст, пол;
- получает стимулы из назначенного набора;
- порядок стимулов случайный для каждого участника;
- каждый стимул показывается 3 секунды или до ответа;
- участник отвечает клавишей/кнопкой `Y` или `N`;
- после `Y` появляется открытый вопрос;
- по каждому стимулу сохраняется отчетная строка.

## Принятая бесплатная архитектура

1. **Frontend:** статический сайт без сборки: `HTML + CSS + JavaScript`.
   Это бесплатно, просто деплоится и не требует сервера.

2. **Хостинг ссылки:** GitHub Pages.
   Он подходит для бесплатной публикации статического сайта через Git.

3. **База ответов:** Supabase Free.
   Браузер отправляет результаты в таблицу через Supabase REST API.

4. **Стимулы:** сейчас это числа 1-20, позже их можно заменить на фото в `assets/photos/`.
   Список стимулов задается в `data/photo-sets.js`, потому что сайт в браузере не может сам безопасно “прочитать все файлы из папки”.

## Что уже реализовано

- Анкета перед тестом.
- Три группы стимулов: `set-a`, `set-b`, `set-c`.
- Временные числовые стимулы 1-20 для каждой группы.
- Разные ссылки для разных групп через параметр `?set=...`.
- Случайный порядок стимулов внутри выбранной группы.
- Показ каждого стимула по таймеру `3000 ms`.
- Ответы через клавиши `Y`/`N` и экранные кнопки.
- Открытый вопрос после ответа `Y`.
- Запись времени реакции в миллисекундах.
- Блокировка телефонов и планшетов.
- Запись размера экрана и окна браузера в отчет.
- Строки отчета с ID сессии, ID участника, анкетой, ID стимула, ответом и текстом.
- Сохранение в Supabase после завершения сессии.
- Резервный режим `?debug=1` для скачивания CSV при локальной проверке.
- SQL-схема таблицы и запрос отчета.
- Русская инструкция запуска и деплоя.

## Что нужно сделать перед реальным запуском

1. Положить настоящие фотографии в `assets/photos/`.
2. Обновить `data/photo-sets.js`, чтобы `id`, `type: "image"` и `src` соответствовали реальным файлам.
3. Создать проект Supabase.
4. Выполнить `supabase-schema.sql` в SQL Editor.
5. Вставить `supabaseUrl` и `supabaseAnonKey` в `config.js`.
6. Пройти тест локально по ссылке `http://localhost:5173/?set=set-a&debug=1`.
7. Загрузить проект в GitHub.
8. Опубликовать сайт на GitHub Pages.
9. Разослать участникам групповые ссылки:
   - `?set=set-a`
   - `?set=set-b`
   - `?set=set-c`

## Минимальный протокол проверки

Перед рассылкой участникам стоит проверить:

- анкета не пускает без имени, возраста и пола;
- каждая группа открывается по своей ссылке;
- все реальные фотографии загружаются, когда они будут добавлены;
- порядок стимулов меняется между разными прохождениями;
- `Y` открывает текстовый вопрос;
- `N` сразу переводит к следующему стимулу;
- после 3 секунд стимул исчезает, но программа продолжает ждать `Y` или `N`;
- в Supabase появляются строки;
- `reaction_time_ms` выглядит реалистично;
- отчет экспортируется в CSV.

## Возможные улучшения после MVP

- Согласие на участие перед анкетой.
- Анонимный режим без имени, если это потребуется этикой исследования.
- Настраиваемые инструкции перед стартом.
- Автоматический экспорт CSV через админ-страницу.
- Блокировка повторного прохождения в одном браузере.
- Балансировка групп автоматически без ручного `?set=...`.
- Отдельная таблица `participants` и отдельная таблица `trials`, если исследование станет крупнее.
