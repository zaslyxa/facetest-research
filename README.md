# Facetest

Небольшое веб-приложение для исследования узнавания лиц. В проект подключены три набора по 60 фотографий. Участник открывает ссылку своей группы, заполняет анкету, видит стимулы в случайном порядке и отвечает `Y` или `N`. По каждому стимулу сохраняются ID сессии, данные анкеты, ID стимула, ответ, размер экрана и время реакции.

## Бесплатная схема

Рекомендуемый вариант:

1. **Хостинг сайта:** GitHub Pages. Для бесплатного GitHub Free используйте публичный репозиторий. Для этого проекта нужен только статический сайт.
2. **Хранение ответов:** Supabase Free. Приложение пишет строки в таблицу через публичный anon key, а чтение с сайта закрыто политиками RLS.
3. **Фото:** хранить веб-копии прямо в проекте в `assets/photos/...`. Репозиторий лучше держать приватным, но сами фото будут доступны участникам по ссылке, потому что сайт должен их показывать.

Полезные официальные страницы:

- GitHub Pages: https://pages.github.com/
- Supabase pricing: https://supabase.com/pricing

## Структура

- `index.html` - интерфейс анкеты и эксперимента.
- `app.js` - рандомизация, таймер, измерение реакции, запись результатов.
- `config.js` - настройки Supabase и параметров эксперимента.
- `data/photo-sets.js` - главный список трех наборов стимулов; работает даже при открытии `index.html` без локального сервера.
- `data/photo-sets.json` - справочная JSON-копия текущих стимулов.
- `data/pretest-questionnaire.js` - вопросы предтестового опросника и гендерные варианты формулировок.
- `assets/photos/` - оптимизированные веб-копии фотографий: по 60 файлов в `set-a`, `set-b`, `set-c`.
- `tools/build-photo-manifest.js` - генератор манифеста по файлам из `assets/photos/`.
- `supabase-schema.sql` - таблица и политика записи для Supabase.
- `supabase-questionnaire-schema.sql` - обновление существующей базы для хранения опросника.
- `supabase-report-query.sql` - короткий запрос для основной выгрузки.
- `supabase-minimal-report-view.sql` - представление с теми же короткими колонками.

## Как обновить фотографии

1. Подготовьте веб-копии фотографий и положите их в папки:
   - `assets/photos/set-a/`
   - `assets/photos/set-b/`
   - `assets/photos/set-c/`
2. Запустите генератор манифеста:

```powershell
node tools/build-photo-manifest.js
```

3. Проверьте пути:

```powershell
node tools/check-project.js
```

Исходные PNG лежат в папке `фото/`, исключенной из Git. Текущие JPEG-копии сохраняют исходные размеры изображений, но весят меньше для быстрой загрузки перед тестом. Имя файла попадет в отчет как часть ID стимула, поэтому после начала сбора данных его лучше не менять.

## Как раздать разные ссылки группам

После публикации сайта отправляйте участникам разные ссылки:

- Группа 1: `https://YOUR_SITE/?set=set-a`
- Группа 2: `https://YOUR_SITE/?set=set-b`
- Группа 3: `https://YOUR_SITE/?set=set-c`

Если параметр `set` не указан, приложение покажет выбор группы в анкете. Это можно отключить в `config.js`, установив `allowSetChoiceWhenMissingUrl: false`.

Для второго этапа исследования, где участник заполняет только анкетные данные и сразу переходит к фототесту, используйте отдельную страницу `stage2.html`:

- Группа 1, этап 2: `https://YOUR_SITE/stage2.html?set=set-a`
- Группа 2, этап 2: `https://YOUR_SITE/stage2.html?set=set-b`
- Группа 3, этап 2: `https://YOUR_SITE/stage2.html?set=set-c`

## Настройка Supabase

1. Создайте проект в Supabase.
2. Откройте SQL Editor и выполните `supabase-schema.sql`.
3. В Supabase откройте Project Settings -> API.
4. Скопируйте Project URL и public anon key.
5. Вставьте их в `config.js`:

```js
window.EXPERIMENT_CONFIG = {
  supabaseUrl: "https://YOUR_PROJECT.supabase.co",
  supabaseAnonKey: "YOUR_PUBLIC_ANON_KEY",
  supabaseTable: "experiment_responses",
  stimulusDurationMs: 3000,
  interStimulusIntervalMs: 500,
  requireMinimumViewport: true,
  minimumViewportWidth: 760,
  minimumViewportHeight: 520,
  preloadConcurrency: 3,
  initialPreloadCount: 60,
  preloadLookaheadCount: 60,
  supabaseRequestAttempts: 8,
  supabaseRequestTimeoutMs: 15000,
  allowSetChoiceWhenMissingUrl: true,
  showDebugDownload: false
};
```

Публичный anon key можно размещать в браузерном приложении. Защита делается не секретностью ключа, а политиками RLS: в текущей схеме участники могут только добавлять строки, но не читать таблицу через сайт.

Если основная схема уже выполнялась до появления опросника, откройте SQL Editor и один раз выполните только `supabase-questionnaire-schema.sql`.

## Опрос перед тестом

Перед демонстрацией стимулов участник проходит 66 обязательных вопросов:

- 3 шкальных вопроса перед методиками;
- 35 утверждений методики 1;
- 28 утверждений методики 2.

В таблицу `experiment_sessions` сохраняется одна строка на участника: введенный ID, возраст, пол, вуз/факультет, параметры экрана, группа и ответы опросника в `questionnaire_answers`.

Незавершенная попытка сохраняется в браузере после каждого шага. После обновления страницы участник продолжит с сохраненного места; при необходимости можно нажать `Начать заново`.

Каждый ответ также сразу отправляется в Supabase в фоне и помещается в локальную очередь до подтверждения записи. Если соединение пропало, приложение не останавливает прохождение: очередь переживает обновление страницы, повторно отправляется при восстановлении сети и проверяется каждые 30 секунд. На финальном экране приложение делает усиленную повторную отправку и безопасно игнорирует уже сохраненные строки.

Если `localStorage` недоступен, приложение предупредит участника не обновлять страницу и всё равно попробует отправлять данные напрямую. Ответы, отправленные до обновления, сохранятся в базе, хотя автоматически продолжить тест с того же места браузер не сможет.

Если Supabase не примет итоговые ответы после повторных попыток, браузер автоматически скачает аварийный CSV-файл и оставит кнопку повторного скачивания на экране. Участник может отправить файл исследователю вручную. Новый CSV содержит не только ответы на фотографии, но и вуз/факультет с ответами предтестового опросника: отдельно по колонкам для просмотра и целиком в JSON-поле для восстановления строки анкеты в базе.

Формулировки автоматически выбираются по полу. Для `female` используется женская версия; для `male`, `other` и `prefer_not_to_say` - мужская.

## Как получить отчет

В Supabase откройте SQL Editor и выполните `supabase-report-query.sql`. Результат можно экспортировать в CSV из интерфейса Supabase.

Основная таблица хранит больше технических полей, чтобы потом можно было восстановить детали процедуры. Для анализа используйте короткий отчет.

Колонки отчета:

- ID испытуемого
- размер экрана
- ID стимула
- ответ `Y` или `N`
- время реакции в миллисекундах

Если хотите иметь готовую таблицу-представление в Supabase, выполните `supabase-minimal-report-view.sql`. После этого можно открывать `experiment_report_minimal`.

## Локальный запуск

Самый простой вариант: откройте файл `index.html` двойным кликом. Приложение не требует локального сервера. Также можно двойным кликом открыть `start-local.bat`.

Если хотите открыть через локальный адрес, можно запустить сервер:

Откройте терминал в папке проекта и запустите:

```powershell
python -m http.server 5173
```

Затем откройте:

```text
http://localhost:5173/?set=set-a&debug=1
```

`debug=1` показывает кнопку скачивания CSV после прохождения. Без настроенного Supabase это удобно для проверки логики.
Также `debug=1` разрешает локальную проверку в узком окне Codex без ограничения минимального размера экрана.

Перед публикацией можно проверить манифест и пути к изображениям:

```powershell
node tools/check-project.js
```

## Деплой бесплатно через GitHub Pages

1. Загрузите проект в GitHub.
2. Откройте Settings -> Pages.
3. Выберите ветку и папку root.
4. Сохраните настройки.

Для GitHub Pages путь сайта может быть вида `https://USER.github.io/REPO/`. Ссылки групп тогда будут `https://USER.github.io/REPO/?set=set-a`.

Минимальные команды Git после создания пустого репозитория на GitHub:

```powershell
git init
git add .
git commit -m "Initial Facetest app"
git branch -M main
git remote add origin https://github.com/YOUR_USER/YOUR_REPO.git
git push -u origin main
```

## Этика и персональные данные

Приложение собирает имя, возраст, пол и ответы участника. Перед реальным исследованием лучше добавить текст информированного согласия и убедиться, что у вас есть право использовать и показывать фотографии. Если имя не нужно для анализа, безопаснее заменить его на код участника.

## Что можно легко изменить

- Длительность показа: `stimulusDurationMs` в `config.js`.
- Число одновременно загружаемых фотографий: `preloadConcurrency` в `config.js`.
- Число попыток записи в Supabase: `supabaseRequestAttempts` в `config.js`.
- Список групп и стимулов: `data/photo-sets.js`.
- Скачивание CSV на финальном экране: `showDebugDownload` в `config.js`.
